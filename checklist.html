Power BI Speed Checklist

By Varun Chaubey • BI Architect (MBA Finance & Marketing)

01 Data Modeling & Star Schema

  • Fact & Dimension Separation: Never use a single flat table. Group dimensions (products, clients) around fact tables.
  • Disable Auto Date/Time: Turn off "Auto date/time" in Options. It creates hidden calendars that bloat file sizes.
  • Bidirectional Filtering: Avoid bidirectional relationships. Use single directional filters to prevent query loops.

02 Advanced DAX Optimization

  • Use DIVIDE(): Never use / for division. DIVIDE() handles division by zero automatically and is faster.
  • Store with variables (VAR): Reuse calculation results instead of rewriting formulas. Saves massive CPU cycles.
  • Use KEEPFILTERS(): Instead of overriding filters, keep context filters active for faster visual calculations.

03 Data Size Compression

  • Remove Cardinality Columns: Delete row IDs, GUIDs, and full timestamp columns if they are not needed for metrics.
  • Prefer Integers over Text: Convert text columns to ID integers. Power BI compresses numbers much faster than strings.
  • No Calculated Fact Columns: Avoid calculated columns in Fact tables. Build them in Power Query or SQL instead.

04 Power Query & Warehousing

  • Enable Query Folding: Perform operations close to source. Verify the "View Native Query" option is active.
  • SQL Joins: Avoid merges/joins in Power Query. Perform joins directly at the Database or Data Lakehouse level.
  • Incremental Refresh: Setup incremental updates for tables exceeding 1 Million rows to save server load.
Pro Tip: Before publishing, run your model through DAX Studio. It shows exact column memory consumption and locates optimization bottlenecks in seconds.
